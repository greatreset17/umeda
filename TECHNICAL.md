# UMEDA — One Unbroken Piece 技術説明書

単一ファイル `index.html`(約1,390行・依存はCDNのThree.js r160のみ)で、大阪・梅田メガハブ全域
—— 淀屋橋・中之島の川辺から淀川対岸(十三)まで —— を「継ぎ目のない1つの探索可能オブジェクト」
として描画するWebGL体験。外部データ・画像・モデルは一切使わず、全て手続き生成。

- 実行方法: `index.html` をブラウザで開くだけ(import mapでunpkgからThree.jsを取得するため要ネット接続)
- 動作要件: WebGL2対応ブラウザ。devicePixelRatioは2にクランプ
- 操作: ドラッグ=オービット / 右ドラッグ=パン / ホイール=ズーム / `P`=フォトモード / `Esc`=解除

---

## 1. 設計思想

参照元「Billion-Piece Manhattan」プロンプトの哲学をそのまま踏襲している。

1. **最初のフレームで全域を見せる**(ロード画面・演出なし。街は「ただそこにある」)
2. **ズームそのものがスペクタクル**(神視点→街路レベルまで、ロード境界もポッピングもない連続ズーム)
3. **10億ピースの「印象」を、生の物量ではなくインスタンシング規律で作る**
4. **完全性は品質設定で絶対に劣化させない**(落とすのは反射・群衆・窓バリエーションのみ)

## 4. 心臓部: patchWindows(ワールド空間・手続き窓システム)

全ビル・全鉄道車両は `MeshStandardMaterial.onBeforeCompile` への注入で窓を得る。
ジオメトリは単位立方体のまま、コストはフラグメントシェーダのみ。

仕組み:

1. 頂点側で `instanceMatrix` 込みのワールド座標・ワールド法線・インスタンス乱数 `vRand` をvaryingで渡す
2. フラグメント側で法線から壁面の「横軸」を選択(`|n.x|>|n.z|` なら Z 軸、そうでなければ X 軸)
3. ワールド座標を窓セルサイズ(ビル: 3.1×3.9m / 電車: 1.6×1.4m)で割ってセルIDを得る
4. `hash(セルID + vRand)` が **その窓固有の点灯閾値** になる
5. 点灯確率は `mix(base, duskP, pow(uDusk,1.3))` —— `uDusk` が上がると閾値の低い窓から
   **1窓ずつ順番に点灯**する。これが「window by window に灯る夕暮れ」の実装
6. 消灯窓はガラス面としてわずかに暗く(diffuse減衰)、点灯窓は暖色/寒色2系統の emissive
7. `roughnessmap_fragment` にも注入し、`vRand` で粗さを0.45〜1.65倍に揺らす
   → 同じマテリアルでも鏡面ギラつきがタワーごとに異なり「二つとして同じ塔がない」

インスタンス描画では `aRand`(InstancedBufferAttribute)、ランドマークの個別メッシュでは
`uRand`(ユニフォーム)と、乱数の渡し方だけ分岐する(`inst` オプション)。

## 5. 地面 = Canvas 2枚

- **albedo**(`gc`): ベースアスファルト→主要面(河川・線路スレート・阪急扇形分岐・御堂筋・
  プラザ・公園)→地区ループ内で街区ごとの歩道色、の順に描画。PlaneGeometry(3200×3400)に
  `SRGBColorSpace` + anisotropy 8 で貼る。回転 `-π/2`・`flipY=true` の組で
  「キャンバス上端 = 北(ZMIN)」になるよう `cz()` を定義している
- **emissive**(`ec`): 街路グリッド線(アンバー)、主要道路の太いストローク、駅コンコースの光溜まり、
  ネオン街4矩形(茶屋町/北新地/堂山/十三)へのHSLランダム色ブロブ。`emissiveIntensity` を
  `0.05+1.9·dusk²` で駆動 → 夜になると街路網が光の網として浮かぶ
- **雲の影**: 地面マテリアルに2オクターブ値ノイズを注入し、`uTime` で流して diffuse を最大20%減光
  (夜は無効化)。ジオメトリ・テクスチャ追加なしの疑似クラウドシャドウ

**排他ゾーン `EXCL`**: 河川・線路回廊・幹線・公園・全ランドマーク足元(+8mマージン)の矩形リスト。
手続きビルは街区矩形がこのリストと重なると生成されない。地理の「正直さ」はこの仕組みで守られる。

## 6. 手続き都市(セクション7)

地区定義(抜粋):

| 地区 | ピッチ | 高さ | パレット | 看板率 |
|---|---|---|---|---|
| 茶屋町 | 46m | 16–64m | 商業ミックス | 0.60 |
| 中崎町 | 26m | 4–11m | 木材・トタン | 0.05 |
| 北新地 | 38m | 12–38m | 商業ミックス | 0.65 |
| 堂島 | 60m | 36–105m | ガラス | 0.06 |
| 船場(淀屋橋) | 56m | 24–68m | 石張り | 0.06 |
| 十三 | 34m | 5–16m | 低層商店 | 0.50 |
| halo-*(枠外) | 95m | 4–26m | 淡色 | 0 |

生成ループ: 街区→1〜4区画に分割→区画ごとに幅・奥行き・高さ(`pow(R(),1.6)` で低層寄りの分布、
`tall` 確率で1.8〜2.8倍のタワー化)→ `BLD[]` へ。同時に高さ24m超なら屋上機械 `CLUT[]`、
看板率に応じ壁面/屋上看板 `SIGNS[]`、150m超なら航空障害灯 `WARN[]` も積む。

描画は:
- **ビル全棟 = InstancedMesh 1個**(単位ボックス+`aRand`属性+instanceColor、影あり)
- 屋上機械 = InstancedMesh 1個(遠距離でフェードアウトする中間LOD層)
- 看板 = InstancedPlane 1個(MeshBasicMaterial。`material.color` のスカラーを
  `0.42+2.6·dusk` で全看板一括増光 —— シェーダ改造なしのネオン)
- 地下街の示唆 = 駅前・御堂筋沿いの「光る階段口」スラブ(同様に色スカラー駆動)
- 街路樹 = 幹/樹冠の InstancedMesh 2個(御堂筋イチョウ並木は細長スケール、公園は広葉)

枠外は `halo-*` 地区(粗ピッチ・低層)が地平線を埋める **インポスター層**。

## 7. ランドマーク(セクション6)

すべて基本形状の組立て+§4のガラス/石マテリアル。代表例:

- **梅田スカイビル**: ツインタワー+水平トーラスの空中庭園リング(夕暮れ発光登録)+
  タワー上部からリングへ斜めに渡る円筒エスカレーターチューブ2本。チューブ内は歩行者経路(§9)
- **HEP FIVE**: 建屋屋上から突き出す赤い観覧車。トーラスリム+スポーク8本のハブが回転し、
  ゴンドラ20個は**非回転の親グループ側**で毎フレーム角度配置(常に直立)。リムは `neon()` 登録で
  夕暮れに赤いネオンリング化
- **大阪ステーションシティ**: ホームスラブ+`rotation.x=0.09` で南下がりに傾けた大屋根グループ
  (半透明ガラス板+横リブ25本+縦通し材+南端エッジのネオンストリップ)+ラチス柱列。
  ノースゲート・ルクア双子・大丸・グランヴィア・駅西も配置
- **百貨店要塞**: 阪急(コーニス付き石張りブロック+187m阪急グランドビル)と阪神+タワーが
  線路を挟んで対峙
- **中之島**: フェスティバルタワー双塔(頂部ランタン発光)、市役所・図書館ドーム、対岸に日銀ドーム

`neon(mat,color,k)` に登録したマテリアルは tick で `emissiveIntensity = 0.12+2.3·dusk·k` に一括駆動。

## 8. 水面(セクション8)

河川・運河・映り込み池は **ShaderMaterial 1個を共有**。UVではなくワールドXZで波を計算するため
どの平面に貼ってもスケールが揃う。内容: 正弦3波の疑似法線 → フレネルで深色⇔空色(`uRefl`=現在の
地平線色)をブレンド → ハーフベクトルによる太陽スペキュラ(指数180)→ 時間量子化ハッシュの
**グリッター斑**(スペキュラを最大7倍ブースト=「金箔を散らした川面」)→ 夜は暖色の街明かり斑を追加。
`fog:true` + fogチャンク手動includeでシーンの霞と整合させ、`tonemapping/colorspace` チャンクも
includeして組込マテリアルと色空間を揃えている。

## 9. 鉄道 —— 梅田の血流(セクション9)

### 路線
CatmullRom曲線(centripetal)。複線は `offsetPts()` でXZ法線方向に±オフセットした2本を生成。

| 路線 | 色 | 編成×本数 | モード | 特記 |
|---|---|---|---|---|
| 阪急3線(神戸/宝塚/京都) | マルーン | 8両×6 | shuttle | 十三停車。**固定の停車秒数で3線の同時発車(三複線並走)が保たれる** |
| JR京都線 | 銀 | 10両×6 | wrap | 大阪駅停車、東淀川側から淀川橋梁 |
| 大阪環状線 | オレンジ | 8両×4 | wrap | 大阪駅・福島停車、西で南へカーブ |
| 阪神本線 | 橙×クリーム交互 | 6両×4 | wrap | **地下(y=−7)から西で地上に浮上**し自前の橋梁で渡河 |
| 東海道新幹線 | 白 | 14両×2 | wrap | 対岸を高速通過(vmax 72m/s)する遠景の白い矢 |

### 状態機械 `updateTrain`
- run: 進行方向の次の停車点を検索し、制動距離 `√(2a·d)` で速度上限を絞る等加速度モデル。
  停車点8m以内は最低這行速度1.4m/sを保証(漸近停止しないための実装上の要点)
- dwell: 停車(中間駅9s / 終端は路線ごとの固定値)。shuttleは終端で `dir` 反転=ターミナル折返し
- wrap: 編成長+マージンを出切ったら反対端へ再投入(画面端の霞の中で入れ替わる)

### 描画
- **全230両 = InstancedMesh 1個**(instanceColorで路線色・阪神の交互色)。車両にも §4 の
  窓シェーダ(小セル・高点灯率)→ 遠景では「光の虫」、近景では窓の並ぶ編成
- 先頭白色灯・尾灯赤 = 列車ごとに1個ずつのインスタンス(進行方向 `dir` で自動入替)
- 曲線に沿って `buildWay()` が高架橋デッキ(26mセグメント)・橋脚(52mごと)を自動生成。
  `inWater(z)` 判定で水上区間だけ **X形トラス+上弦材** に切替。y<1.2(阪神地下区間)はスキップ
- 架線柱+ビーム: 駅周辺(|x|<680, z −680〜380)のみ生成し、近距離LODでフェードイン

道路系高架(阪神高速2本・新御堂筋・十三大橋)も同じ `buildWay()` を使い回す。十三大橋は
さらに正弦アーチ2本+吊り材を `pushBar()` で装飾。

## 10. 道路交通と群衆(セクション10–11)

- 道路 = ポリライン+レーン配列(`{off, dir}`)。御堂筋は4レーン全て南行・タクシー率45%
- 車720台。毎フレーム `s += speed · wave · dt`、`wave = 0.55+0.45·sin(0.021s − 1.35t + φ)`
  → 位置依存の速度波が自然な **圧縮=ブレーキリップル** を作る(信号・車間ロジック不要)
- 描画はボディ/前照灯/尾灯/タクシー行灯の4 InstancedMesh。`count` プロパティで密度スライダーと
  品質係数ぶんだけ先頭から描画(行列更新も打ち切るので削減がそのままCPU/GPU削減になる)
- 前照灯類は「dusk>0.18 または カメラ650m以内」でのみ更新・表示
- バス5台が駅前ロータリーの閉ループを周回
- 歩行者: 15経路(駅デッキ・HEPスクランブル・グラングリーン対角線・中之島遊歩道・
  スカイビルのエスカレーターチューブ内!)を往復移動+上下ボブ。経路ごとの
  `0.6+0.4·sin(0.45t+φ)` パルスで見える人数が呼吸=信号サイクルの暗示

## 12. 光と時間(セクション2+14)

3プリセット(golden/noon/dusk)は目標値 `tgtL` を差し替えるだけで、毎フレーム
`k = 1−e^(−1.15·dt)` の指数補間で `cur` が追従 → 空・太陽・霞・露出・`uDusk` が数秒かけて連続変化。
`uDusk` ひとつが下流の全て(窓・看板・ネオン・街路発光・前照灯・星・水面の街明かり)を駆動する
**単一時刻変数アーキテクチャ**。dusk遷移中に窓が1つずつ灯っていくのは§4のハッシュ閾値の副産物であり、
追加コードは存在しない。

## 13. LODとパフォーマンス(セクション13–14)

- **距離クロスフェード**(ポッピング対策): 屋上機械 1600→2600m / 架線 500→850m /
  歩行者 750→1100m / 湯気 520m / ゴンドラ 1600m。透明マテリアルの opacity を
  smoothstepで滑らかに落としてから非表示化
- **品質セレクタ**: High(dpr≤2, 影2048, 樹冠影あり)/ Balanced(dpr≤1.5, 影1024, 交通75%)/
  Light(dpr1, 影なし, 交通50%, `uLow=1`)。**街の完全性とランドマークシルエット
  (リング・赤い観覧車・大屋根)はどのティアでも削らない**
- 静的ジオメトリは全て構築時に行列確定(以後 `needsUpdate` なし)。動的なのは
  列車・車・歩行者・警告灯の行列書き換えのみ(`DynamicDrawUsage`)
- ドローコールは数十本。総インスタンス数 ≈ ビル1.2万 + 屋上機械 + 看板 + 樹木 + 高架部材 + 動体
- `frustumCulled=false` を全InstancedMeshに設定(インスタンス群のバウンディング誤カリング防止)

## 15. 既知の割り切り

- 地理は「知識からの説得力ある近似」であり測量精度ではない(プロンプト要件どおり)
- 阪急3線の三複線同時発車は路線長差により長時間経過で位相がずれる(初期数サイクルは同期)
- エスカレーター内歩行者の横オフセットはチューブ径に対しやや広い
- 影マップ1枚(2048px)で全域をカバーするため、超近接では影の解像度が粗い
- Osaka Metro御堂筋線は非表示(地下)。「光る階段口」で存在を示唆するに留める
