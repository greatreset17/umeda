# UMEDA — One Unbroken Piece

単一のHTMLファイル（約1,390行・依存はCDNのThree.js r160のみ）で、大阪・梅田メガハブ全域（淀屋橋・中之島の川辺から淀川対岸の十三まで）を「継ぎ目のない1つの探索可能オブジェクト」として描画するWebGLプロシージャル都市環境です。外部の3Dモデル、テクスチャ画像、地理データファイルは一切使用せず、すべて数理アルゴリズムによる手続き生成で構築されています。

本プロジェクトは、大規模言語モデル（LLM / Fable 5）における高度な空間推論力、すなわち「インターネット上のテキスト知識から、実在する都市の立体構造（XYZ数理座標、メッシュ変形、LOD、アニメーション、環境エフェクト）を脳内で再構築し、実行可能な最適化WebGLコードへ直接翻訳する能力」の技術検証を目的としています。

## 🛠 実行方法・動作要件

### 実行方法
`index.html` を任意のWebブラウザで開くだけで動作します。
※ import map を用いて CDN（unpkg）から Three.js を動的に取得するため、インターネット接続が必要です。

### 動作要件
- WebGL2 対応ブラウザ
- `devicePixelRatio` は最大2にクランプ（パフォーマンス最適化のため）

### 基本操作
- **ドラッグ**: オービット（視点回転）
- **右ドラッグ**: パン（平行移動）
- **ホイール**: 連続シームレスズーム
- **`P` キー**: フォトモード（UI非表示）
- **`Esc` キー**: フォトモード解除

## 📐 設計思想と技術的特徴

1. **初期フレームの即時性**
   ロード画面や段階的な明かし（Reveal）演出を排除。神視点（God View）から全域が最初の一フレームで「ただそこにある」状態を実現。
2. **ズームスペクタクルとLOD規律**
   超高空からの鳥瞰から、窓枠のディテールや走行車両が見える街路レベルまで、ロード境界やポッピング（オブジェクトの突発的な出現）のない完全な連続ズームを達成。
3. **Brutal Instancingとパフォーマンス**
   1万2千棟を超えるビル、数千の屋上機械、720台の車両、630体の群衆パルスを、少数の `InstancedMesh` に集約。静的ジオメトリは構築時に行列を確定させ、動体（列車・自動車等）のみ `DynamicDrawUsage` で毎フレーム更新。ドローコールを数十本に抑制。

## ⚡ コア・アルゴリズム詳解

### 1. ワールド空間・手続き窓システム（patchWindows）
全ビル・全鉄道車両は、`MeshStandardMaterial.onBeforeCompile` へのGLSL注入によりファサードテクスチャを得ています。ジオメトリ自体はシンプルな単位立方体のまま、コストをすべてフラグメントシェーダー内に閉じ込めています。
頂点から渡されたワールド座標をセルの実寸サイズ（ビル: 3.1×3.9m / 電車: 1.6×1.4m）で除算してセルIDを算出し、`hash(セルID + インスタンス固有乱数)` を元に各窓の点灯閾値を決定。時刻変数 `uDusk` の上昇に伴い、閾値の低い窓から **「1窓ずつ個別に（window by window）」灯っていく** 動的な夕暮れを実装。さらに `roughness` もインスタンスごとに揺らすことで、「二つとして同じ鏡面反射を持たない塔」を表現しています。

### 2. 鉄道・高架の動的モデリングシステム（rail / buildWay）
CatmullRom曲線から生成された複線に対し、`buildWay()` が高架橋デッキ（26mセグメント）および橋脚（52mピッチ）を動的生成。Z軸座標による `inWater(z)` 判定をリアルタイムに行い、淀川をまたぐセグメントのみ **X形鋼トラス構造および上弦材** へとプロシージャルに切り替えます。阪神本線の地下から地上への浮上（y=-7 からの遷移）もこのマニフォールド内で処理されます。

### 3. 位置依存の速度波による交通流（traffic）
720台の車両の挙動は、個別の衝突判定ロジックを持たず、`wave = 0.55 + 0.45 · sin(0.021s - 1.35t + φ)` という数式ベースの速度波によって制御されています。これにより、道路の特定座標に差し掛かった車両が自律的に減速・密集し、自然な **圧縮およびブレーキランプのリップル（波形効果）** が発生します。

## 📊 品質管理とLOD（詳細度）

都市の完全性と主要ランドマークのシルエット（空中庭園のリング、HEPの赤色、大阪駅大屋根）は、最低動作ティアでも絶対に喪失・破壊されないようガードレールが敷かれています。

- **距離クロスフェード**: 屋上機械（1600→2600m）、架線柱（500→850m）、歩行者（750→1100m）を、`smoothstep` を用いたマテリアル不透明度（opacity）の減衰によってポッピングなしで滑らかに消去。
- **品質セレクタ（UI）**: 
  - **High**: dpr=2, シャドウマップ 2048px, 樹冠影有効
  - **Balanced**: dpr=1.5, シャドウマップ 1024px, 交通量 75%
  - **Light**: dpr=1, 影マップ完全無効化, 交通量 50%, フラグメント内負荷削減（`uLow=1`）

## 📝 免責・割り切り
- マクロな地理関係（駅、線路のファンアウト、河川、大通り）はプロンプトの制約に従い「知識からの説得力ある近似」として構築されており、実測測量データ（GIS等）のインポートは行っていません。
- 阪急3線の三複線同時発車システムは、各路線のCatmullRom曲線長の差異により、長時間稼働時に位相のズレが生じます（初期数サイクルは完全同期）。
